# DELIVERABLES/

Home for **cross-cutting one-off deliverables** that aren't tied to a
single project — strategy, research, and analysis documents the Operator
commissioned that don't belong to one repo (e.g. a portfolio-wide
constraint analysis, a cross-cluster research report), plus personal
deliverables that aren't notes and aren't repo-bound code. Anything
scoped to one repo belongs in that repo (`PROJECTS/<org>/<repo>/`);
personal notes and knowledge belong in the Obsidian vault
(`NOTES/MASTER/`). Filing rule: if you can name the one project repo a
file belongs to, it does NOT go here — move it.

This folder is tracked in git so deliverables travel across machines.
Naming: `kebab-case.md` (or a native extension like `.docx` for
client-bound output).
